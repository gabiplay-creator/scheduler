# 공정표 — 인테리어 공사 스케줄러

빌드 과정 없이 브라우저에서 바로 동작하는 단일 HTML 파일(React, CDN 로드)로 만든 인테리어 공사 스케줄러입니다.

## 기능
- **현장관리**: 현장(주소) 등록, **전체 공사 시작일~종료일** 등록, 관리사무소 이름/연락처 등록, 도면/견적서 업로드(이미지·PDF·엑셀) 및 보기·다운로드, 현장별 공사 기간 관리
  - 견적서를 엑셀(.xlsx/.xls)이나 CSV로 올리면 앱에서 표로 바로 미리보기가 가능합니다
  - 공정별(철거·목공 등) 일정은 시작일~종료일(총 기간)로 별도 등록합니다
- **공사캘린더**: 두 가지 보기 모드 지원
  - **달력형**: 현장마다 서로 다른 색의 전체 공사기간 막대가 그 현장의 공정 막대들 바로 위에 그룹으로 묶여서 표시됩니다. 같은 "목공"이라도 어느 현장 것인지 헷갈리지 않도록 현장이 2곳 이상 보일 땐 막대에 현장명이 함께 표시돼요. 그 달에 실제로 진행되는 공정만 표시되어 빈 줄이 길게 늘어지지 않습니다
  - **전체 목록형**: 전체 공사기간과 모든 공정 일정을 시작일순으로 한 화면에서 확인 (월 이동 없이 전체 스크롤)
  - 두 모드 모두 현장별 필터 지원
- **담당자관리**: 철거·샷시·목공·타일·도기·전기·가구·도배·도배지·마루·중문·도장·필름·입주청소·메지 15개 공정에 기본 담당자가 미리 등록되어 있고, 공정별로 담당자를 자유롭게 추가/삭제/변경 가능
  - **+ 공정 추가**로 목록에 없는 새 공정(카테고리)을 이름/메모/기본 담당자와 함께 추가할 수 있고, 색상은 기존 공정들과 겹치지 않게 자동으로 배정됩니다
  - 공정 자체를 삭제할 수도 있습니다 (관련 일정도 함께 삭제됨)
- **거래처관리**: 14개 공정에 속하지 않는 자재상·조명업체 등 기타 협력업체 정보를 자유롭게 추가/수정/삭제 가능 (기본으로 "여기가 우리집(박과장)", "한빛조명" 등록되어 있음)

## 실행 방법
빌드가 필요 없습니다. `index.html`을 더블클릭해서 브라우저로 열면 바로 사용할 수 있습니다.

## GitHub에 올리기
```bash
git init
git add .
git commit -m "인테리어 공사 스케줄러 초기 버전"
git branch -M main
git remote add origin <레포지토리 URL>
git push -u origin main
```

## GitHub Pages로 배포하기
1. GitHub 저장소 Settings → Pages 이동
2. Source를 `main` 브랜치, 루트(`/`) 폴더로 설정 후 저장
3. 몇 분 후 `https://<아이디>.github.io/<레포지토리명>/` 로 접속 가능

## 데이터 저장 방식 (중요)
이 앱은 별도 서버 없이 **브라우저의 localStorage에만 데이터를 저장**합니다.
- 같은 브라우저에서는 새로고침해도 데이터가 유지됩니다.
- 다른 기기나 다른 브라우저에서는 데이터가 공유되지 않습니다.
- 브라우저 저장공간(보통 5~10MB)에 한계가 있어, 도면/견적서 이미지는 너무 큰 파일(4MB 이상)은 업로드가 제한됩니다.
- 여러 명이 함께 쓰거나 기기 간 동기화가 필요하다면, 추후 Firebase/Supabase 같은 백엔드를 연결해 localStorage 부분만 교체하면 됩니다.

## 커스터마이징
- 기본 담당자 정보는 `index.html` 상단의 `DEFAULT_TRADES` 배열에서 수정할 수 있습니다.
- 공정별 캘린더 색상도 같은 배열의 `color` 값을 바꾸면 반영됩니다.

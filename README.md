# 공정표 — 인테리어 공사 스케줄러

빌드 과정 없이 브라우저에서 바로 동작하는 단일 HTML 파일(React, CDN 로드)로 만든 인테리어 공사 스케줄러입니다.

## 기능
- **현장관리**: 현장(주소) 등록, **전체 공사 시작일~종료일** 등록, 관리사무소 이름/연락처 등록, 도면/견적서 업로드(이미지·PDF·엑셀) 및 보기·다운로드, 현장별 공사 기간 관리
  - 현장 카드는 **전체 공사 시작일이 빠른 순서로 자동 정렬**됩니다. 새 현장을 추가하면 그 시작일에 맞는 위치에 자동으로 끼워 넣어져요
  - 카드를 **드래그해서 순서를 직접 바꿀 수도 있습니다** — 한 번 드래그로 옮긴 순서는 계속 유지됩니다
  - 견적서를 엑셀(.xlsx/.xls)이나 CSV로 올리면 앱에서 표로 바로 미리보기가 가능합니다
  - 공정별(철거·목공 등) 일정은 시작일~종료일(총 기간)로 별도 등록합니다. 등록된 일정 칩을 **클릭하면 수정 창**이 열립니다 (✕ 버튼은 바로 삭제)
- **공사캘린더**: 세 가지 보기 모드 지원
  - **달력형**: 이번 달과 다음 달을 **하나로 이어진 달력**으로 보여줘서(요일 칸이 끊기지 않음), 월말/월초에 걸친 일정도 자연스럽게 이어서 파악됩니다 (‹›로 1개월씩 이동). 날짜 옆에 요일이 작게 같이 표시되고, 토요일은 파란색·일요일은 빨간색으로 구분됩니다. **날짜 하나하나 기준으로 정확하게** 새 달이 시작하는 칸에만 남색 선이 표시되고, 같은 줄 안에서도 그 달에 속한 날짜만 은은한 배경색이 깔립니다(예: 7월 마지막 주에 8월 1일이 섞여 있어도 7/26~31은 흰색, 8/1만 베이지색). 각 달의 칸들 한가운데에는 옅은 음각 숫자(7, 8 등)로 몇 월인지 표시됩니다. **주(週)마다 칸 위쪽 경계선 색이 파란빛/베이지빛으로 번갈아** 표시되어, 하루에 여러 공정이 쌓여도 어느 줄이 이번 주이고 다음 줄이 다음 주인지 확실히 구분됩니다. 현장마다 서로 다른 색의 전체 공사기간 막대가 상단에 표시되고, 그 아래로 **그날 실제로 진행되는 공정만** 표시됩니다(공사명 반복 없이 "철거", "목공"처럼 간단히). 공정이 없는 날은 빈 줄을 만들지 않아 여백이 늘어지지 않습니다
    - 공정 막대 색상 표시를 **"배경색"(꽉 채운 색상) / "글자색"(색깔 글자, 배경·테두리 없음)** 두 가지 중 선택할 수 있습니다 — 색이 눈에 잘 안 들어오면 글자색 모드로 바꿔보세요
    - **달력 왼쪽에 현장명을 포스트잇처럼 고정 표시**합니다 — 예전엔 공정이 없는 빈 날에 현장명 글씨가 매번 들어가서 헷갈렸는데, 이제 그 자리엔 색상 배경만 남고 "어느 현장 줄인지"는 왼쪽 라벨만 보면 바로 알 수 있어요. 그 주(週)에 전혀 일정이 없는 현장은 라벨 자체가 표시되지 않습니다
    - 일정에 **품수(0.5 단위)**와 **작업자명(직접 입력)**을 넣으면, 각 현장 블록의 공정 3칸 **바로 아래에 그날 배정된 작업자 이름들이 한 줄로 모여서** 표시됩니다(예: "👷 반장, 정수" — 여러 공정에 각각 다른 작업자를 넣어도 한 줄로 합쳐서 보여줍니다). 이 담당자 줄을 **클릭하면 그날 진행되는 공정별로 작업자명을 바로 지정**할 수 있는 창이 열립니다. "담당자관리" 탭의 품수 합계는 이 **작업자명 기준**으로 집계됩니다(작업자명을 안 넣으면 담당자 거래처 이름으로 대체)
    - 공정 막대를 **클릭하면 수정 창이 열립니다** (시작일/종료일/담당자/품수/메모 변경 가능, 삭제 버튼도 여기 있음)
    - 현장 블록 안의 **빈 칸을 클릭하면 그 현장·그 날짜로 바로 새 공정을 추가**할 수 있습니다 (현장관리 탭까지 안 가도 됨)
    - **공사 기간 밖의 완전히 빈 날짜**(항상 옅게 "+ 일정 추가"가 보이고, 마우스를 올리면 더 진해짐)를 클릭해도 일정을 추가할 수 있습니다 — 현장을 직접 선택해서, 착공 전 자재 주문·실측 방문 같은 미리 챙길 일정을 등록할 수 있어요
    - 공정 막대를 **다른 날짜로 드래그하면 일정이 그 날짜로 이동**합니다 (여러 날짜짜리 일정은 기간을 유지한 채 통째로 옮겨집니다)
  - **현장별 보기**: 한 번에 **현장 하나의 달력만** 크게 보여줍니다. 상단 현장 필터에서 원하는 현장을 선택하면 그 현장으로 바로 전환되고, 다른 현장 달력이 아래에 같이 나오지 않습니다. 이번 달·다음 달 두 달이 아래로 이어서 표시됩니다
  - **전체 목록형**: 전체 공사기간과 모든 공정 일정을 시작일순으로 한 화면에서 확인 (월 이동 없이 전체 스크롤)
  - 세 모드 모두 현장별 필터 지원 — **여러 현장을 동시에 체크/해제할 수 있는 다중 선택**이라, 3개 중 원하는 2개만 골라서 보는 것도 가능합니다 (체크박스 목록, "전체 선택"/"전체 해제" 버튼 포함)
- **거래처관리** (구 담당자관리): 철거·샷시·목공·타일·도기·전기·가구·도배·도배지·마루·중문·도장·필름·입주청소·메지 15개 공정에 기본 거래처(담당자) 정보가 미리 등록되어 있고, 공정별로 자유롭게 추가/삭제/변경 가능
  - **+ 공정 추가**로 목록에 없는 새 공정(카테고리)을 이름/메모/기본 담당자와 함께 추가할 수 있고, 색상은 기존 공정들과 겹치지 않게 자동으로 배정됩니다
  - 공정 자체를 삭제할 수도 있습니다 (관련 일정도 함께 삭제됨)
- **기타거래처** (구 거래처관리): 14개 공정에 속하지 않는 자재상·조명업체 등 기타 협력업체 정보를 자유롭게 추가/수정/삭제 가능 (기본으로 "여기가 우리집(박과장)", "한빛조명" 등록되어 있음)
- **담당자관리** (신규): 일정에 입력한 **품수(0.5 단위)**를 월 단위로 공정별·담당자별 합산해서 보여주는 리포트입니다. ‹›로 월을 이동하면서 확인할 수 있고, **일당 계산**할 때 참고하시면 됩니다
- **담당자 일정** (신규): 날짜(행) × 담당자(열) 표 형식으로, 선택한 담당자가 그날 어느 현장에 있는지 한눈에 볼 수 있습니다
  - 상단에서 **보고 싶은 담당자만 체크**해서 선택할 수 있습니다 (전체 선택/해제 가능)
  - 각 셀은 그 담당자가 그날 배정된 **현장 색상**으로 표시되고, 현장명 앞 두 글자가 표시됩니다 (예: "홍은", "두산") — 아래 범례에서 전체 현장명을 확인할 수 있어요
  - **작업자명 칸에 "이대희, 손정수"처럼 쉼표로 여러 명을 한 번에 입력하면 자동으로 한 명씩 나눠서** 담당자 목록·품수 집계·담당자 일정표에 각각 반영됩니다
  - ‹›로 월 이동, 토요일/일요일 행은 색으로 구분됩니다. **이번 달·다음 달 두 달치가 한 번에** 표시됩니다 (날짜는 "7/20"처럼 월/일로 표기). **표 머리글(날짜/요일/담당자명)은 스크롤해도 상단에 고정**되어 계속 보입니다
  - 선택한 담당자 모두 일정이 없는 날짜는 **자동으로 숨겨집니다** — 단, 일정이 있는 첫 날부터 마지막 날 사이에 낀 빈 날짜는 흐름이 끊기지 않도록 그대로 보여줍니다(그 앞뒤의 완전히 비어있는 날짜만 잘라냅니다). 필요하면 "빈 날짜도 표시" 체크박스로 전체 날짜를 다시 볼 수 있어요

## 실행 방법
빌드가 필요 없습니다. `index.html`을 더블클릭해서 브라우저로 열면 바로 사용할 수 있습니다.

## GitHub에 올리기
```bash
git init
git add .
git commit -m "인테리어 공사 스케줄러 초기 버전"
git branch -M main
git remote add origin <레포지토리 URL>
git push -u origin main
```

## GitHub Pages로 배포하기
1. GitHub 저장소 Settings → Pages 이동
2. Source를 `main` 브랜치, 루트(`/`) 폴더로 설정 후 저장
3. 몇 분 후 `https://<아이디>.github.io/<레포지토리명>/` 로 접속 가능

## 데이터 저장 방식 (중요)
기본 상태에서는 별도 서버 없이 **브라우저의 localStorage에만 데이터를 저장**합니다.
- 같은 브라우저에서는 새로고침해도 데이터가 유지됩니다.
- 다른 기기나 다른 브라우저에서는 데이터가 공유되지 않습니다.
- 브라우저 저장공간(보통 5~10MB)에 한계가 있어, 도면/견적서 이미지는 너무 큰 파일은 업로드가 제한됩니다.
- 여러 명이 함께 보고 실시간으로 같이 수정하려면, 바로 아래 안내대로 무료 Firebase를 연결하세요.

## 여러 명이 함께 보고 실시간으로 수정하기 (Firebase 연동)
기본 상태에서는 이 앱이 각자의 브라우저에만 데이터를 저장합니다. **다른 사람과 함께 보고, 누가 수정하면 바로 반영되게** 하려면 무료 Firebase 프로젝트를 하나 만들어 연결하면 됩니다. (구글 계정만 있으면 됩니다, 신용카드 등록 불필요)

1. https://console.firebase.google.com 접속 → **프로젝트 추가**로 새 프로젝트 생성
2. 왼쪽 메뉴 **빌드 → Firestore Database** → **데이터베이스 만들기** → 위치 선택 후 **테스트 모드로 시작** 선택
3. Firestore Database 화면에서 **규칙(Rules)** 탭을 열고 아래 내용으로 바꾼 뒤 **게시**:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read, write: if true;
       }
     }
   }
   ```
   (링크만 있으면 누구나 보고 고칠 수 있게 여는 규칙입니다 — 팀 내부용 링크이니 외부에 함부로 공유하지 않는 걸 권장해요. 나중에 로그인 기능이 필요하면 알려주세요.)
4. 프로젝트 설정(톱니바퀴 아이콘) → **일반** 탭 → 맨 아래 "내 앱"에서 **웹 앱 추가**(</> 아이콘) → 앱 등록
5. 화면에 나오는 `firebaseConfig` 객체 값을 복사
6. `index.html` 파일을 열어 `FIREBASE_CONFIG` 부분(파일 상단 근처)을 복사한 값으로 교체
7. 저장 후 커밋/푸시

```bash
git add index.html
git commit -m "Firebase 실시간 공유 연결"
git push
```

적용되면 화면 맨 아래 표시가 "🟢 LIVE SYNC ON"으로 바뀌고, 그 순간부터는:
- 같은 링크에 접속한 모든 사람이 **같은 데이터**를 봅니다
- 누군가 현장을 추가하거나 일정을 바꾸면 **다른 사람 화면에도 바로** 반영됩니다
- 별도 로그인 없이 링크만 알면 누구나 보고 수정할 수 있습니다

**참고**: 실시간 공유가 켜지면 도면/견적서 이미지 업로드 용량 제한이 650KB로 줄어듭니다(공유 데이터 저장 공간의 문서당 용량 제한 때문). 너무 큰 이미지는 압축하거나 캡처본으로 올려주세요.

Firebase를 아직 연결하지 않았다면 지금까지처럼 이 브라우저에만 저장되는 방식으로 계속 정상 작동합니다 — 연결은 언제든 나중에 해도 됩니다.
